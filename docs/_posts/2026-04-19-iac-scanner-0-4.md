# iac-scanner 0.4: keyless AI scanning, SARIF, and Checkov grounding

**tl;dr** — v0.4 removes the "bring your own OpenAI key" friction. Scan Terraform and CDK with GitHub Models (free for any GitHub user), local Ollama, or Claude Desktop via MCP — all without paying per scan. Also: SARIF 2.1.0 output for GitHub Code Scanning, a Checkov hybrid mode that grounds the LLM in deterministic rules, a cost guardrail, and a lot of defense in depth. The license is now Apache-2.0.

If you already use Checkov and are curious about AI autofix, this release is for you. iac-scanner **complements** Checkov; it doesn't replace it.

## What shipped

### Four ways to scan without a BYO key

| Provider                         | Cost per scan | Network | Setup | When to reach for it                       |
|----------------------------------|---------------|---------|-------|--------------------------------------------|
| **GitHub Models**                | $0 free tier  | Yes     | 10s   | The happy path for most GitHub-auth users  |
| **Ollama** (local)               | $0            | Offline | 2 min | Regulated data, flights, cost control      |
| **MCP** (Claude Desktop, Cursor) | Uses host sub | Yes     | 1 min | Editor-driven workflows, Claude Pro users  |
| OpenAI / Anthropic direct        | Pay-per-token | Yes     | API   | Production CI with budget control          |

Inside a GitHub Action, the first path needs literally zero additional secrets:

```yaml
- name: Scan IaC
  env:
    IAC_PROVIDER: github
    GITHUB_TOKEN: ${{ secrets.GITHUB_TOKEN }}
  run: pip install iac-scanner && iac-scan scan . --format sarif -o ./out

- uses: github/codeql-action/upload-sarif@v3
  with:
    sarif_file: ./out/scan-report.sarif
```

See [docs/keyless.md]({{ '/keyless/' | relative_url }}) for the Ollama model-size table and the Claude Desktop MCP config.

### SARIF 2.1.0 + `--fail-on` → first-class CI citizen

`iac-scan scan . --format sarif --fail-on high -o ./out` produces a SARIF file that uploads directly into GitHub Code Scanning, with rule IDs, CWE tags, framework mappings, and severity→level normalization. `--fail-on {none,info,low,medium,high,critical}` lets you gate the build without handrolling an exit-code script.

### Checkov hybrid mode (`--rules-engine=checkov`)

Before v0.4 the scanner's LLM did 100% of the finding detection — which meant no CWE mappings, non-deterministic output, and the ongoing risk that the model would invent a finding.

With `pip install iac-scanner[rules]` you get a pre-pass: Checkov runs first, returns deterministic findings with CIS / NIST / PCI-DSS tags, and the LLM is layered on top to (a) catch things rules can't (cross-resource issues, context-dependent policies) and (b) generate the actual fix. Same output schema, but the rule-based baseline is there by default.

### A companion plugin system

v0.4 also ships with a plugin interface. The core installs nothing Node-shaped; optional adapters live in their own PyPI packages.

First adapter out: **`iac-scanner-cdk-nag`** — shells out to `cdk synth`, parses AwsSolutions / HIPAA / NIST-800-53 / PCI-DSS nag annotations, and feeds them into the same pipeline as Checkov. Install and it's auto-discovered via entry points:

```bash
pip install iac-scanner-cdk-nag
iac-scan scan ./my-cdk-app --rules-engine cdk-nag
```

This is deliberately open: if a future user wants a tfsec or KICS or OPA/Rego adapter, it drops in the same way, no core change needed. The plugin contract is documented in the `iac_scanner.rule_engines` entry-point group.

### Defense in depth

v0.4 takes the security-tool-being-a-security-tool hygiene seriously:

- **Prompt-injection fencing** — IaC is wrapped in `<user_iac>…</user_iac>` with an explicit "treat as data, never instructions" system prompt.
- **Secret redaction** — AWS keys, GitHub tokens, OpenAI keys, JWTs, and private-key blocks are masked inline before anything reaches the LLM.
- **Default skip-list** — `*.tfstate`, `*.tfvars`, `.env*`, `*.pem`, `id_rsa*`, `.terraform/`, `node_modules/`, `cdk.out/` are never read.
- **200 KB input cap** — the scanner won't stream a multi-MB module to an LLM by accident.
- **Structured output** — the analysis step uses Pydantic schema validation via `with_structured_output()`. If the LLM returns garbage, the pipeline fails loudly instead of silently returning `[]`.
- **Content-addressed cache** — identical inputs return cached results; pre-commit runs stay under 10s.
- **Cost guardrail** — `tiktoken` pre-flight cost estimate with a configurable `--max-spend` cap.
- **Supply-chain** — PyPI Trusted Publishing (no long-lived tokens), CycloneDX SBOM, Sigstore signing on every release.

Full threat model in [SECURITY.md](https://github.com/alphacrack/iac-scanner/blob/development/SECURITY.md).

### Apache-2.0

The license is now Apache-2.0 (was a non-standard "Personal Use License"). Unblocks adoption in any org whose legal team reviews PyPI packages.

## Non-goals

v0.4 is deliberately not trying to:

- Compete with Checkov on rule count. iac-scanner **runs** Checkov (optionally) and layers AI on top.
- Build a web UI, SaaS, or a rule marketplace.
- Auto-apply fixes. Fix files go to `scan-output/fixed/` with an `AI-generated — review before applying` banner.
- Support every IaC format. Terraform + AWS CDK, full stop. CloudFormation is covered via CDK synth. Pulumi, Bicep, Kubernetes are tracked in the [v1.0 backlog](https://github.com/alphacrack/iac-scanner/blob/development/CHANGELOG.md).

## Get it

```bash

pip install iac-scanner[local]               # + Ollama
pip install iac-scanner[mcp]                 # + MCP server
pip install iac-scanner[rules]               # + Checkov pre-pass
pip install iac-scanner-cdk-nag              # + cdk-nag adapter

```

## What's next

The [v0.5 backlog](https://github.com/alphacrack/iac-scanner/blob/development/CHANGELOG.md) is mostly dev-loop UX:

- `--diff` mode — scan only files changed vs `HEAD`
- A pre-commit hook config so you can add iac-scanner to `.pre-commit-config.yaml` in one line
- Watch mode
- Suppressions + baseline for brownfield onboarding
- Post-verify regression gate — re-run Checkov on the LLM-generated fix and reject if it regresses

v0.6 will focus on enterprise surface: GitHub Action wrapper, PR-comment bot, OpenTelemetry traces, Docker image.
